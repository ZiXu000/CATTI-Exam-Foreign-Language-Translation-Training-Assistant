# 【Code with SOLO】零基础也能造出硬核翻译应用：用 SOLO 搭建全功能 CATTI 备考神器

## 1. 摘要
用 TRAE SOLO 打造了一款功能完整的 CATTI（全国翻译专业资格考试）助考助手。不仅支持笔译的智能批改与得分点提取，还创新性地实现了口译的全真模拟（通过大模型将听力文本自动转化为判断、选择及语义分块练习，并接入 Xiaomi MiMo TTS 生成语音），完美解决了高昂的翻译人工批改和缺乏模拟口译环境的痛点。

## 2. 背景
我是一名备考 CATTI 的翻译爱好者（也是对 AI 技术感兴趣的开发者），深知翻译备考中的两大痛点：一是笔译练习如果没有专业老师批改，很难发现自己的翻译盲点和得分漏区；二是口译练习缺乏互动性，单纯听录音看原文无法模拟真实的考场高压，且找不到合适的音频素材和配套考题。

我希望能有一个专属的 AI 助考助手，能自动为我批改笔译，甚至能把随便一段文本变成带有语音的口译实战考卷。然而，独立开发这样一个包含前端复杂答题交互、后端数据库、大模型 Prompt 编排、语音 TTS 处理的全栈 Web 应用，以我个人的精力原本需要数周甚至数月。于是，我决定拉上 TRAE SOLO 试试。

## 3. 实践过程
整个项目的开发就像是和一位经验丰富的全栈架构师结对编程。

**任务拆解：**
- **笔译模块**：输入原文和译文，通过大模型生成带具体得分点的批改报告。
- **口译生成器**：分为“口译综合能力”（生成判断题、选择题、摘要要求）和“口译实务”（按意群语义切分），并接入 TTS 实现音频播放。
- **前端交互与历史记录**：打造沉浸式的考试界面，做题时隐藏原文只听语音，交卷后红色高亮错题、绿色高亮正确答案，并将所有记录和录音缓存存入 SQLite 数据库。

**用了 SOLO 的哪些能力：**
- **全栈项目搭建**：通过对话让 SOLO 帮我用 React + Tailwind CSS 搭好前端，用 FastAPI 搭好后端。
- **复杂逻辑处理**：大模型结构化 JSON 输出解析、交互式判卷逻辑（根据用户点击实时反馈）。
- **主动踩坑与纠错**：解决前端状态流转、音频生命周期和组件布局的问题。

**中间踩过什么坑（翻车实录）：**
- **JSON 解析翻车**：最初让大模型返回考卷数据时，后端正则表达式过滤太严格，导致有效的 JSON 键被破坏。把报错贴给 SOLO 后，它迅速定位到了问题并重写了安全且兼容性更好的解析正则。
- **TTS 语音生成 500 报错**：接入小米 MiMo TTS 时，发现一旦传入的转录文本过长，API 就会直接返回 500 错误。
  - *解决方式*：我让 SOLO 帮忙重构，利用正则表达式（`(?<=[.!?。！？\n])\s+`）将长文本优雅地切分为不超过 200 字符的短句分别请求 TTS，再用 Python 原生的 `wave` 模块从底层字节级别将音频帧无缝拼接起来。完美解决了大段文本无法生成语音的报错问题！
- **TTS 请求太慢且浪费额度**：每次查看历史记录或者重复听题都要重新请求语音，既慢又费钱。
  - *解决方式*：提出需求后，SOLO 建议并实现了 TTS 缓存机制（`ttsCache`），将生成的 base64 音频存入字典，并随考卷一起落库保存到 SQLite。二次加载历史考卷时实现音频零延迟播放、零 API 调用。

## 4. 成果展示
最终成果是一个极具现代感的响应式 Web 应用：

- **笔译模式**：沉浸式双栏输入，倒计时沙漏，自动比对，生成细致到“词汇”的批改报告。
- **口译模式**：输入一段文字，秒变完整考卷。在“考试中”状态，左侧的文本原文会自动收起隐藏，只露出一个居中悬浮的语音播放按钮，杜绝“视译”作弊。用户可以直接在生成的选项或意群文本框中作答，交卷后直观展示对错（绿对红错）。
- **历史归档**：所有的考试记录、用户填写的译文、交互作答记录、甚至 TTS 语音缓存，都被完整保留，随时可以复盘。

**技术栈一览：**
- **前端**：React 18 + TypeScript + Tailwind CSS + Vite + Lucide Icons
- **后端**：FastAPI + SQLAlchemy + SQLite
- **AI 能力**：DeepSeek (负责结构化批改与逆向生成考卷) + Xiaomi MiMo TTS (负责高质量语音合成)

## 5. 效果与总结

**提效了多少？**
原本预估需要 1 到 2 个月的全栈开发周期（光是搞定音频流拼接解码和前端复杂的交互布局就能让人头疼很久），在 TRAE SOLO 的加持下，仅用了几个晚上的业余时间就跑通了全部闭环，并且拥有极高的完成度和 UI 质感。

**SOLO 在流程中做了什么？**
它不仅仅是一个“代码生成器”，更是一个能听懂产品需求并给出工程化方案的“技术合伙人”。比如当我提出“口译模式下默认隐藏文本框只露出播放按钮”时，它直接利用 Tailwind 帮我写出了一个优雅的悬浮收起/展开侧边栏；遇到 TTS 报错，它能立刻拿出纯 Python 字节流合并音频帧的底层解决方案。

**有没有可复用的方法？**
- **Prompt 逆向工程**：让大模型扮演考官时，明确约束“生成极具迷惑性的错误选项（如语音混淆、时空倒置、主语替换）”，这让自动生成的考卷质量极高，不再是“人工智障”。
- **让 AI 解决 AI 的限制**：遇到大模型接口（如 TTS）的长度限制时，不要气馁，直接把报错扔给 SOLO，让它写文本切片和音频重组代码，这是解决大模型 API 限制的万能公式。

在这个 AI 时代，TRAE SOLO 赋予了我们把“脑洞”瞬间变现的超能力。无论是备考党、学生还是普通开发者，只要有想法，你也能轻松造出属于自己的超级利器！
